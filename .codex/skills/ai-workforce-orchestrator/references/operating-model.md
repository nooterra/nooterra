# Operating Model

## Cadence

- Sprint length: 2 weeks.
- Planning window: 12-18 weeks rolling.
- Milestone review: every 2 sprints.

## Role responsibilities

- PM: scope, prioritization, acceptance criteria.
- Tech Lead: architecture and sequencing.
- Backend: APIs, persistence, workers.
- Frontend: buyer/operator workflow surfaces.
- QA: verification plans, regression safety.
- DevOps: release safety, observability, infra scale.
- GTM: pilot conversion, adoption metrics.

## Exit criteria per sprint

- Committed tickets completed >= 80%.
- No critical regressions in verification path.
- One measurable customer-facing improvement shipped.
