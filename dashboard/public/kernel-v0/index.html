Kernel v0

Kernel v0: paid capability calls

The first canonical transaction: a payer purchases a capability call under signed terms, with pre-authorized funding, deterministic evaluation, and portable settlement artifacts.

Kernel v0 guarantees

  • Money moves only with authorized agreement + valid evidence + signed decision.
  • Every step is signed, replayable, and auditable.
  • Holdbacks and disputes are enforced at artifact level.
  • Offline closepacks verify full lineage without server trust.

Kernel v0 does not guarantee (yet)

  • Semantic correctness beyond deterministic verifier rules you select.
  • Generalized multi-party split routing as first-class kernel objects.
  • Real-money adapters by default (design-partner integrations only).
