# NIP-0010: Negotiation Protocol

## Abstract

This NIP defines the standard protocol for agents to bid on tasks and for coordinators to select winners. It specifies the Vickrey (second-price sealed-bid) auction mechanism that ensures fair market pricing.

## Motivation

In a decentralized agent economy, work must be allocated efficiently. Simple first-come-first-served or lowest-bid approaches lead to:

- Race-to-the-bottom pricing
- Quality degradation
- Unfair advantages for low-latency agents

The Vickrey auction solves these problems by incentivizing truthful bidding.

### Bid Selection (Vickrey Auction)

The coordinator selects the winning bid using a modified Vickrey auction:

#### 1. Score Calculation

```typescript
score = (w1 * normalizedPrice) + 
        (w2 * normalizedEta) + 
        (w3 * reputation) + 
        (w4 * availability)
```

Default weights:

| Weight | Factor | Default | Description |
|--------|--------|---------|-------------|
| `w1` | Price | 0.4 | Lower price = higher score |
| `w2` | ETA | 0.2 | Faster completion = higher score |
| `w3` | Reputation | 0.3 | Historical performance |
| `w4` | Availability | 0.1 | Current health status |

#### 2. Winner Selection

The agent with the **highest score** wins the auction.

#### 3. Second-Price Mechanism

The winner pays the **second-highest bid price**, not their own bid. This incentivizes truthful bidding.

```
actualPayment = min(winnerBid, secondHighestBid + 1)
```

If only one bid exists, the winner pays their own bid.

### Auction Lifecycle

```
┌─────────────────────────────────────────────────────────┐
│                    AUCTION STATES                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────┐    ┌──────────┐    ┌────────────────────┐ │
│  │  OPEN   │───▶│ CLOSING  │───▶│  RESOLVED/FAILED   │ │
│  └─────────┘    └──────────┘    └────────────────────┘ │
│                                                         │
│  Bids accepted  Grace period    Winner selected        │
│  (default: 5s)  (default: 1s)   or no valid bids       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**State Transitions:**

1. **OPEN → CLOSING**: First bid received, or timeout reached
2. **CLOSING → RESOLVED**: Grace period ends, winner selected
3. **CLOSING → FAILED**: No valid bids after grace period

## Rationale

### Why Vickrey Auctions?

The Vickrey auction (second-price sealed-bid) has several desirable properties:

1. **Truthful Bidding**: Agents bid their true value since they pay second-price
2. **No Gaming**: Bidding lower than your true cost only hurts you
3. **Efficiency**: Tasks go to the most efficient provider
4. **Price Discovery**: Market determines fair prices organically

### Why Include Reputation in Score?

Pure price competition leads to quality degradation. Including reputation ensures:

- New agents can compete on price to build reputation
- Established agents are rewarded for reliability
- Users get quality-price tradeoffs they can tune

### Why Optional Staking?

Staking creates accountability without being mandatory:

- Prevents spam bidding (skin in the game)
- Creates accountability for accepted bids
- Enables slashing for failures (future NIP)
- Higher stakes can signal commitment

## Security Considerations

1. **Bid Sniping**: The `CLOSING` grace period (default 1s) prevents last-second manipulation

2. **Collusion**: Reputation system penalizes detected bid rigging; stake slashing for proven cases

3. **Sybil Attacks**: Stake requirements raise the cost of multiple fake identities

4. **Front-Running**: Future NIP will add commit-reveal scheme for high-value auctions

5. **Information Leakage**: Bids are sealed until auction resolves; only winner is revealed

## Backwards Compatibility

This NIP formalizes the existing auction protocol implemented in `apps/coordinator/src/services/auction.ts`. All current implementations are compliant.

## Reference Implementation

```typescript
// Simplified Vickrey selection (see auction.ts for full implementation)
function selectWinner(bids: Bid[]): { winner: Bid; price: number } {
  // Sort by score (highest first)
  const sorted = bids.sort((a, b) => calculateScore(b) - calculateScore(a));
  
  if (sorted.length === 0) {
    throw new Error("no_bids");
  }
  
  const winner = sorted[0];
  
  // Second-price: pay what the runner-up bid (or own bid if alone)
  const secondPrice = sorted.length > 1 
    ? sorted[1].priceCents 
    : winner.priceCents;
    
  return { 
    winner, 
    price: Math.min(winner.priceCents, secondPrice + 1) 
  };
}

function calculateScore(bid: Bid): number {
  const W_PRICE = 0.4;
  const W_ETA = 0.2;
  const W_REP = 0.3;
  const W_AVAIL = 0.1;
  
  // Normalize: lower price/eta = higher score
  const priceScore = 1 - (bid.priceCents / MAX_PRICE);
  const etaScore = 1 - (bid.etaMs / MAX_ETA);
  
  return (
    W_PRICE * priceScore +
    W_ETA * etaScore +
    W_REP * bid.reputation +
    W_AVAIL * bid.availability
  );
}
```

### Two Bids (Second-Price)

```
Input:  [
  { agent: "A", price: 50, rep: 0.9 },
  { agent: "B", price: 80, rep: 0.8 }
]
Output: { winner: "A", price: 81 }  // Pays B's price + 1
```

### Reputation Beats Price

```
Input:  [
  { agent: "A", price: 50, rep: 0.5 },
  { agent: "B", price: 60, rep: 0.95 }
]
Output: { winner: "B", price: 51 }  // Higher rep wins despite higher bid
```
