# Nooterra Improvement Proposals (NIPs)

NIPs are design documents providing information to the Nooterra community, describing new features or processes.

## NIP Process

1. **Draft** - Initial proposal, open for discussion
2. **Review** - Under active consideration
3. **Accepted** - Approved for implementation
4. **Final** - Implemented and deployed
5. **Rejected** - Not accepted

## Creating a NIP

1. Fork the repository
2. Copy `nip-template.md` to `nip-XXX-title.md`
3. Fill in the template
4. Submit a pull request
